ADDR_AdminAcc_TEST

public with sharing class ADDR_AdminAcc_TEST
Tests specific to Administrative Account Address Management.

Salesforce.org
2016

Methods

  • clearAddrExisting
    Update accounts' billing address fields to empty cause clearing the default address verify: existing Default Address updated to be non-default no new Address created
  • configSettings
  • insertMultilineStreetAddress
    Insert account w/ multiline street address verify: new default Address created and the street address is broken up into street and street2
  • insertNewDefaultAddr
    Insert new default addresses to Account w/ existing default addresses verify: Account address matches new default old default addresses no longer marked default
  • insertNewDefaultAddrAndReset
    Insert new default addresses to Account with existing default addresses, and then go back verify: Account address matches original default new default addresses no longer marked default
  • insertNewDefaultMultilineStreetAddr
    Insert new default multiline street addresses w/ existing default addresses verify: Account address matches new default street & street2 combined into contact and account single street field
  • newAddrNoParent
    Create an Address with no Account verify: runtime error
  • newDupeAddr
    Create N new duplicate Addresses from Account verify: duplicates not created
  • newDupeAddrNonDefault
    Create N new duplicate Addresses from Account, with non-default existing verify: duplicates not created found match becomes Default
  • setAddrOverride
    Set account address override. verify: account's address matches override
  • updateAddrExisting
    Update accounts' billing address to cause an update of the default Address object. verify: existing Default Address updated Account get updated Address
  • updateAddrExistingCaseOnly
    Update Account's billing address with just a case change, to cause an update to default Address object verify: existing Default Address updated Account has updated Address
  • updateAddrMarkOverride
    update Address' billing address and mark it as override, to cause creation of new non-default Address object verify: new Address created original Addresses still default
  • updateAddrNew
    Update contacts' mailing address to cause creation of new default Address object. verify: new Default Address created Accounts get new Address
  • updateDefaultAddr
    Update existing default Addresses. verify: account billing address fields updated
  • updateMultilineStreetAddr
    Update accounts' billing address with multiline street address verify: new Default Address created ACcounts get new Address new Addresss has multiline street split into Street and Street2 fields
  • updateNonDefaultAddr
    Making an Address non-default. That is, the address is no longer the default one for the Account. verify: no change to account address
